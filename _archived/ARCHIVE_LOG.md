本目录用于存放已弃用但需要保留的功能模块和文档。

---

## 目录结构

```
_archived/
├── features/      # 弃用的功能模块（代码）
├── docs/          # 过期的文档
└── ARCHIVE_LOG.md # 本文件：归档日志
```

---

## 归档规范

### 归档时机
- 功能被替代或重构时
- 文档内容过期但有参考价值时
- MVP 简化过程中移除的功能

### 归档流程
1. 将文件移动到对应子目录
2. 在本日志中记录归档信息
3. 更新所有引用该文件的链接
4. 提交 Git 并注明归档原因

### 记录格式
```
## YYYY-MM-DD 归档标题

**归档文件：** `原路径 → _archived/xxx/`
**归档原因：** 简述为什么移除
**替代方案：** 新功能/文档的位置（如有）
**操作人：** 谁执行的归档
```
