# 超协体专用任务列表使用说明

## 🎯 功能概述

超协体项目现已配置**专用任务列表**，通过环境变量 `CLAUDE_CODE_TASK_LIST_ID=supercoordination` 实现任务隔离。

## ✅ 已完成的配置

### 1. 启动脚本已更新
文件：`~/super.sh`
```bash
#!/bin/bash
# 设置超协体专用任务列表
export CLAUDE_CODE_TASK_LIST_ID=supercoordination
cd ~/ClaudeWorkspace/supercoordination-mcp && ./start-server.sh
```

### 2. 快捷命令已配置
在 `~/.zshrc` 中：
```bash
alias super='~/super.sh'
```

## 📋 任务列表功能

### 什么是专用任务列表？

每个 `CLAUDE_CODE_TASK_LIST_ID` 对应一个独立的任务存储空间：

| 任务列表 ID | 用途 | 示例任务 |
|-------------|------|----------|
| `supercoordination` | 超协体项目 | 成员接入、功能开发、文档更新 |
| `groceries` | 购物清单 | 买菜、日用品采购 |
| `work` | 工作项目 | 工作任务、会议待办 |
| (不设置) | 默认通用 | 临时任务 |

### 好处

✅ **任务隔离**：超协体的任务不会和其他项目混在一起
✅ **自动切换**：启动超协体项目时自动使用专用列表
✅ **清晰管理**：不同项目的任务分门别类

## 🎓 最佳实践

1. **一个项目一个列表**：为每个重要项目设置专用任务列表 ID
2. **命名规范**：使用小写英文字母和连字符，如 `super-coordination`、`my-project`
3. **定期清理**：完成的任务会自动归档，但可以定期检查清理
4. **备份重要任务**：任务数据在 `~/.claude/task_lists/` 下，可以备份

## ❓ 常见问题

**Q: 如果不设置任务列表 ID 会怎样？**
A: 会使用默认任务列表，所有项目的任务会混在一起。

**Q: 可以在不同终端窗口使用不同的任务列表吗？**
A: 可以！每个终端会话的环境变量是独立的。

**Q: 任务列表数据存在哪里？**
A: `~/.claude/task_lists/<任务列表ID>/tasks.json`

**Q: 如何删除一个任务列表？**
A: 直接删除对应目录：`rm -rf ~/.claude/task_lists/supercoordination/`

**Q: 超协体的任务数据会丢失吗？**
A: 不会，任务数据持久化存储在磁盘上，重启电脑也不会丢失。
